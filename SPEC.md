## 1. プロダクト概要

| 項目 | 内容 |
|---|---|
| アプリ名（仮） | AI科学者メンター |
| コンセプト | 科学技術の偉人と対話し、仕事の姿勢・信念・モチベーションを学ぶ |
| ターゲット | 技術系ビジネスマン（エンジニア・研究者・設計者など） |
| コアバリュー | 「同じ技術者として困難を乗り越えた偉人の言葉は、自己啓発本より響く」 |
| 提供形態 | Webアプリ（スマホ対応レスポンシブ） |

---

## 2. ターゲットユーザーの解像度

**ペルソナ：**
- 28〜45歳の技術系ビジネスマン
- エンジニア・研究者・製品設計者・技術営業など
- 仕事で壁にぶつかることが多い（プロジェクト停滞・理解されない・長期研究の孤独）
- 自己啓発・読書習慣あり → 課金意欲あり

**ユースケース：**
- 「プロジェクトが全然進まない。ファラデーだったらどう考える？」
- 「研究が誰にも評価されない。ガリレオはどう信念を保ったの？」
- 「新しい挑戦が怖い。ニュートンのモチベーションの源泉は？」

## 3. MVP機能定義（2週間スコープ）

### 含めるもの ✅

| # | 機能 | 説明 |
|---|---|---|
| 1 | トップページ | コンセプト説明 + 偉人選択へのCTA |
| 2 | 偉人選択画面 | 8名のカード一覧。名前・分野・キャッチコピー |
| 3 | チャット画面 | 選んだ偉人AIとのリアルタイム対話 |
| 4 | プリセット質問ボタン | 「仕事の壁を相談する」「信念を聞く」「名言を教えて」など3〜4個 |
| 5 | レスポンシブ対応 | スマホ・PC両対応のUI |

### 含めないもの ❌（Phase 2以降）

| 機能 | 理由 |
|---|---|
| ユーザー認証・アカウント | MVPでは不要。まず体験を届ける |
| 会話履歴の保存 | DBなしで開発を最速化 |
| 課金・有料プラン | まず使ってもらうことを優先 |
| 偉人の追加リクエスト機能 | コンテンツで需要を測ってから |
| スマホアプリ（native） | WebのPWAで代替 |

## 5. 画面フロー

```
[トップページ /]
  ↓ 「偉人を選ぶ」ボタン
[偉人選択 /characters]
  ↓ 偉人カードをクリック
[チャット /chat/[scientist-id]]
  ↓ 質問入力 or プリセットボタン
  ↓ Claude APIで回答生成
  ↓ 偉人の言葉として表示
  ↓ 続けて対話 or 別の偉人へ戻る
```

## 6. 技術スタック

| レイヤー | 技術 | 選定理由 |
|---|---|---|
| フロントエンド | Next.js 15 + TypeScript | SSRでSEO対応・個人開発で標準的 |
| スタイリング | Tailwind CSS | 高速UI構築 |
| AIバックエンド | Claude API（claude-haiku-4-5） | 高品質・低コスト・感情表現が自然 |
| API通信 | Next.js API Routes（streaming対応） | サーバーレス・Vercelと相性◎ |
| DB | なし（MVP段階） | 会話はセッションのみ。コスト・工数ゼロ |
| 認証 | なし（MVP段階） | まず使ってもらう |
| ホスティング | Vercel | 無料枠・Next.jsと完全統合 |

## 7. プロンプト設計方針

各偉人のシステムプロンプトに含める要素：

```
1. 時代背景・生涯の概要（箇条書き）
2. 主要な発見・業績
3. 性格・思想・言葉のトーン
4. 仕事への姿勢・信念
5. 代表的な名言（実際の言葉）
6. 回答ルール
   - 一人称は偉人本人として話す
   - 現代の仕事・技術の文脈に置き換えて応用する
   - 歴史的事実に基づく（フィクションはNG）
   - 押しつけず、対話的に
   - 日本語で回答する
```
